# Phishing Detection & Reporting Demo

A full-stack cybersecurity awareness web application that simulates a phishing attack workflow across three roles: **attacker**, **victim**, and **admin**.

## Tech Stack

- Frontend: React + Tailwind CSS + Recharts
- Backend: Python Flask
- ML Model: Pre-trained Gradient Boosting Classifier (`Phishing_model.pkl`)
- Storage: In-memory Python dictionaries (no database)

## Prerequisites

- Python 3.10+
- Node.js 18+
- Place `Phishing_model.pkl` in the `/backend/` folder

## Backend Setup

```bash
cd backend
pip install -r requirements.txt
python app.py
```

The backend runs at **http://localhost:5000**.

## Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Open **http://localhost:5173** to use the app.

## Full Demo Walkthrough

1. Log in as **Attacker**.
2. Compose a phishing email and send it to `victim@demo.com`.
3. Log out and log in as **Victim**.
4. Open an inbox email, then click **Open Website** or **Scan Link Only**.
5. The ML model scans the URL in real-time and blocks phishing sites.
6. If blocked, submit a report using **Report This Website to Admin**.
7. Log out and log in as **Admin**.
8. Review reports, resolve one, and send a confirmation back to the victim.
9. Log back in as **Victim** to see the admin response in the inbox.

## Backend Endpoints

- `POST /login` вЂ” `{ email, password }`
- `GET /inbox/<email>` вЂ” returns received and sent messages
- `POST /send-email` вЂ” send a new phishing email
- `PATCH /email/<id>/read` вЂ” mark an email as read
- `POST /scan-url` вЂ” scan a URL and return prediction details
- `POST /report` вЂ” submit a phishing report
- `GET /reports` вЂ” return all reports
- `PATCH /report/<id>/action` вЂ” resolve a report and notify the victim
- `GET /stats` вЂ” return admin dashboard metrics

## Model & Feature Extraction

The app uses a pre-trained gradient boosting model and expects **21 features** in the following order:

- UsingIP
- PrefixSuffix-
- SubDomains
- HTTPS
- NonStdPort
- HTTPSDomainURL
- RequestURL
- AnchorURL
- LinksInScriptTags
- ServerFormHandler
- InfoEmail
- AbnormalURL
- WebsiteForwarding
- StatusBarCust
- DisableRightClick
- AgeofDomain
- DNSRecording
- WebsiteTraffic
- PageRank
- GoogleIndex
- StatsReport

Feature values are:

- `1` = legitimate
- `-1` = phishing indicator
- `0` = suspicious

## Notes

- All data is stored in-memory; restarting the backend resets state.
- The ML model returns `1 = legitimate` and `-1 = phishing`.
- The app includes CORS for `http://localhost:5173`.
- If `Phishing_model.pkl` is missing, the scan endpoints will be unavailable until the file is added.

## Troubleshooting

- If scans fail, confirm `backend/Phishing_model.pkl` is present.
- If the scaler is missing, run `python build_scaler.py` from `/backend`.
- If CORS errors appear, verify frontend is on port `5173` and backend on `5000`.
